# 05 — Décisions structurantes

Ce dossier est le référentiel des décisions structurantes du projet Chaweer.

## Organisation

### adr/ — Architecture Decision Records

Les ADR sont des documents **historiques et immuables**. Chaque ADR enregistre
une décision d'architecture à un instant T, avec son contexte, les options
étudiées, la décision retenue et ses conséquences.

Une ADR ne se modifie pas : elle est supplantée par une nouvelle ADR.

#### Template ADR

Chaque ADR doit suivre la structure suivante :

- Statut
- Date
- Contexte
- Problème
- Options étudiées
- Décision retenue
- Conséquences
- Alternatives futures

### standards/ — Standards du projet

Les standards sont des documents **vivants** qui définissent les règles de
gouvernance de l'ingénierie et du développement : Definition of Done,
Definition of Ready, conventions de code, conventions Git, conventions de
tests, checklist de release, etc.

Ces documents évoluent avec le projet.
