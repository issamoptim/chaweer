## Description

As a user, I want to authenticate using my phone number (OTP) so that I can access my account without email.

## Rationale for Deferral

Phone is currently read-only in the profile (displayed with lock icon). Phone authentication requires SMS provider integration (e.g., Twilio, local Moroccan provider) which adds cost and complexity. Deferred to Post-MVP Sprint 09. Also blocks US-081 (SMS Notifications).

## Acceptance Criteria (Draft)

- User can register/log in with phone number + OTP
- OTP sent via SMS (requires SMS provider)
- Phone number verified during registration
- Phone becomes editable after verification
- Supports Moroccan phone numbers (+212)
