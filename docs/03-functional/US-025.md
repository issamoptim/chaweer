## Description

As a user who forgot their password, I want to reset it via email so that I can regain access to my account.

## Rationale for Deferral

Critical UX feature, but Google OAuth is the dominant authentication path. Email/password users are secondary. The Forgot Password screen already exists in the auth module (Screen 7) but the full flow (email send, reset link, new password form) is not implemented. Deferred to Post-MVP Sprint 09.

## Acceptance Criteria (Draft)

- User enters email → receives reset link (anti-enumeration: neutral confirmation message)
- Reset link opens a new password form
- Password policy: min 8 characters, at least one letter and one digit
- Reset link expires after a defined period
- Resend option with 30s countdown
