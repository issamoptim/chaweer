## Description

As a user who registered with email/password, I want to verify my email address so that my account is confirmed and secure.

## Rationale for Deferral

Google OAuth already provides a verified email identity for the majority of users (Google is the dominant auth method). Email verification for email/password users is important for trust but not blocking launch. Deferred to Post-MVP Sprint 09.

## Acceptance Criteria (Draft)

- After registration, user receives a verification email with a link
- Clicking the link verifies the email and redirects to the app
- Verified badge displayed on profile
- Resend verification email option
- Expiration and invalid link handling
