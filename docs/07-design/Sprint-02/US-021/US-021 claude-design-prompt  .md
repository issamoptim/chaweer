# Chaweer – UI Design – My Profile

You are the Lead Product Designer for the Chaweer project.

## Existing Project Context

The Authentication module has already been fully designed inside this Claude project.

It defines the visual identity of Chaweer.

Your mission is NOT to redesign the application.

Extend the existing design system.

Everything you create must feel like a natural continuation of the existing product.

Reuse the existing:

- layout
- spacing
- typography
- colors
- border radius
- shadows
- cards
- buttons
- inputs
- switches
- icons
- animations
- interaction patterns

Do not introduce a new visual language.

A user navigating from Authentication to My Profile should feel that both experiences were designed together.

---

# Objective

Design the complete **My Profile** experience for a Public User.

This screen allows users to consult their personal information and manage simple preferences.

Profile editing is handled in another screen and is **not part of this design**.

---

# Information to Display

Display the following information:

- First Name
- Last Name
- Email Address
- Phone Number (if available)
- Country
- City (if available)
- Nationality (if available)
- Preferred Language

Missing optional information should be displayed gracefully.

---

# Language

The application language is automatically detected from the browser.

Users can change it.

Available languages:

- Français
- العربية
- English

The language change should feel immediate and intuitive.

---

# Notifications

Display notification preferences.

Provide independent controls for:

- Email notifications
- Push notifications
- SMS notifications

SMS is not available yet.

Display it as disabled with a subtle "Coming soon" indication.

---

# Delete Account

Provide a dedicated section for account deletion.

This action must remain visually secondary.

It should communicate caution without creating unnecessary anxiety.

---

# Exclusions

Do not include:

- Profile picture
- Avatar upload
- Password management
- Security settings
- Login history
- Activity history
- Billing
- Subscription
- Professional profile
- Theme selector
- Dark mode
- Statistics

---

# UX Requirements

Create a premium SaaS experience.

The interface should feel:

- Modern
- Elegant
- Calm
- Trustworthy
- Minimal

Prioritize readability and simplicity.

Avoid unnecessary decoration.

Respect WCAG AA accessibility guidelines.

Reuse the existing Chaweer Design System.

---

# Deliverables

Produce a complete design including:

- Main profile screen
- Loading state
- Empty state
- Error state
- Success feedback
- Delete account confirmation modal

Describe component behavior and interaction where necessary.

The result must be production-ready for frontend implementation.

---

# Important

Do not invent business rules.

Do not remove required information.

Do not add new features.

Follow the requirements exactly.

Improve the user experience while preserving the intended functionality.

The final result must look like a seamless extension of the existing Authentication module.
