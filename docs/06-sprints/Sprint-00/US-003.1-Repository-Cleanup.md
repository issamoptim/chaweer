# US-003.1 — Repository Cleanup

## Business Value

Garantir un dépôt Git propre, maintenable et conforme aux bonnes pratiques avant de poursuivre le développement de Chaweer.

Cette User Story ne développe aucune fonctionnalité métier.

Elle vise uniquement à nettoyer et fiabiliser le dépôt.

---

# Contexte

Les User Stories US-001, US-002 et US-003 ont permis de mettre en place les fondations du projet.

Une revue d'architecture a identifié que certains fichiers générés ne doivent pas être versionnés.

Cette User Story vise à remettre le dépôt dans un état propre avant de poursuivre le développement.

---

# Objectif

Nettoyer complètement le dépôt Git.

Vérifier que seuls les fichiers sources nécessaires sont versionnés.

---

# Travail demandé

## Git

Vérifier que les éléments suivants ne sont jamais suivis par Git :

- node_modules/
- dist/
- coverage/
- .vite/
- .turbo/
- .cache/
- .DS_Store
- *.log
- .env
- .env.local
- .env.*.local

---

## .gitignore

Vérifier les fichiers :

- .gitignore (racine)
- apps/api/.gitignore
- apps/web/.gitignore

Compléter si nécessaire.

---

## Nettoyage Git

Retirer de Git tous les fichiers générés déjà versionnés.

Conserver les fichiers localement.

Utiliser uniquement :

git rm --cached

Ne jamais supprimer les fichiers physiques nécessaires au développement.

---

## Vérification des packages

Vérifier que :

- package-lock.json est conservé.
- node_modules n'est jamais versionné.

---

## Vérification des builds

S'assurer que les dossiers suivants ne sont pas suivis :

- dist/
- coverage/
- .vite/

---

## Vérification des variables d'environnement

Conserver :

.env.example

Ignorer :

.env

.env.local

.env.production

.env.development

---

## Vérification macOS

Ignorer :

.DS_Store

---

## Vérification IDE

Ignorer les fichiers temporaires de :

- VS Code
- Windsurf
- JetBrains

s'ils ne sont pas utiles au projet.

---

## Contraintes

Ne modifier :

- aucun code métier ;
- aucune User Story ;
- aucun fichier source de l'application ;
- aucune configuration fonctionnelle.

Cette User Story est exclusivement dédiée au nettoyage du dépôt.

---

# Vérifications obligatoires

Exécuter :

git status

git ls-files

npm run build

npm run lint

npm test

Corriger tout problème détecté.

---

# Critères d'acceptation

- Aucun node_modules versionné.
- Aucun dist versionné.
- Aucun .vite versionné.
- Aucun .env versionné.
- Aucun .DS_Store versionné.
- Tous les .gitignore sont corrects.
- Le build fonctionne.
- Les tests passent.
- Le lint passe.

---

# Livrables

- Dépôt Git nettoyé.
- .gitignore mis à jour.
- Historique Git propre.
- Documentation mise à jour si nécessaire.

---

# Définition de Terminé

La User Story est terminée lorsque :

- le dépôt ne contient plus aucun fichier généré ;
- tous les .gitignore sont conformes ;
- le projet compile ;
- les tests passent ;
- le lint passe ;
- aucune modification métier n'a été effectuée.
